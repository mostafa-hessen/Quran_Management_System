## NON-NEGOTIABLE RULES
--------------------------------------------------

# 1️⃣ Clean Separation of Concerns

STRICTLY separate:

- UI Components
- Business Logic
- API Calls
- State Management
- Validation
- Types
- Utilities

Never place logic inside JSX if it should be in hooks/services.

--------------------------------------------------

# 2️⃣ Feature-Based Development

If request belongs to attendance:

Put code inside features/attendance/

If auth:

features/auth/

If shared reusable component:

shared/components/

Never place code in wrong module.

--------------------------------------------------

# 3️⃣ TypeScript Discipline

Always create proper types/interfaces.

Rules:

- No any
- No implicit any
- Reusable types
- Shared types in shared/types
- Feature types inside feature/types

--------------------------------------------------

# 4️⃣ React Query Standards

Use React Query for server state.

Rules:

- queries in hooks/
- api calls in api/
- proper query keys
- invalidate queries after mutations
- loading states
- error states

Never place fetch directly inside component unless tiny exception.

--------------------------------------------------

# 5️⃣ Zustand Standards

Use Zustand only for global client state:

- auth state
- sidebar state
- filters if global

Never use Zustand for server fetched data.

--------------------------------------------------

# 6️⃣ React Hook Form Standards

All forms must use:

- react-hook-form
- reusable controlled inputs
- validation schema if needed

--------------------------------------------------

# 7️⃣ UI / MUI Rules

Use Material UI professionally.

STRICT RULES:

- Use theme palette only
- Use spacing from theme
- Use typography from theme
- Use borderRadius from theme
- Use shadows from theme

Never hardcode:

- colors
- random padding
- random font sizes

Bad Example:
background: "#333"

Good Example:
backgroundColor: theme.palette.primary.main

--------------------------------------------------

# 8️⃣ Design Quality Rules

UI must feel modern SaaS quality.

Required:

- clean spacing
- hierarchy
- elegant cards
- responsive
- loading skeletons
- empty states
- smooth UX
- professional Arabic RTL support if Arabic used

Never generate ugly/basic UI.

Can this be reused?
